# Async vs. Sync
- In Java, an https request is synchornous and the program will halt to wait for the request to complete. 
- In javascript, which is async, the code keeps executing as the https request (I/O) occurs. 
- This is also because javascript is single threaded and if we halt for an I/O operation, the whole program becomes non-interactive until the operation finishes. See more on javascript's event loop, callback queue, stack, webAPI.(Runtime execution context).


- Multi threading is possible with browsers using webAPI, web workers.

# Axios and Promises
- _A Promise is an object representing the eventual completion or failure of an asynchronous operation._
- we use `.then` function to define event handlers for a promise.


# Effect Hooks
- Effect is basically an eventHandler (loosely) that occurs simply because of rendering of a component.
- It is mainly for synchronizing components with external systems.
- Whether we get data successfully or not, the component will render and the useEffect function will execute after render. 

- Dependency array is used as the second parameter to only execute the useEffect function when certain values change.
