
# REST
-   In REST terminology, we refer to individual data objects, such as the notes in our application, as _resources_. Every resource has a unique address associated with it - its URL. According to a general convention used by json-server, we would be able to locate an individual note at the resource URL _notes/3_, where 3 is the id of the resource. The _notes_ URL, on the other hand, would point to a resource collection containing all the notes.

- Creating a new resource for storing a note is done by making an HTTP POST request to the _notes_ URL according to the REST convention that the json-server adheres to. The data for the new note resource is sent in the _body_ of the request. json-server requires all data to be sent in JSON format. What this means in practice is that the data must be a correctly formatted string and that the request must contain the _Content-Type_ request header with the value _application/json_.

# Notes when building the country search app:


- When i did `setQuery` and then used the query to build a regex, I forgot that the state may or may not be updated yet since the function hasn't ended. The correct way is to use the value that will be put in the state. (Here it was `e.target.value`)

- **Are props individual to each component instance in an array?** Yes. But props don't persist — they come from the parent. If you need a value that survives re-renders or changes from user interaction, store it as state inside the component.

- **Must state handlers be defined inside the component?** Yes. An external function has no way to reference a specific instance's state.

- **What happens if a prop and state variable share the same name?** The state shadows the prop, making the prop inaccessible. Rename one of them.

- **What happens if a country is deleted from the server between polls?** React unmounts that component instance cleanly — no crash. State is destroyed. Danger zone is if you're mid-request fetching details about that country when it gets deleted — handle errors there. Also, if data changes between polls, state initialized from a prop won't update because `useState(initialValue)` only uses the initial value once.
