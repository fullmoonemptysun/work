- npm: tool that manages javascript packages, does dependency handling and also runs scripts like `npm run dev.
- `npm init` to create a Node.js project in the current directory. (This is not a React project neccessarily).
- Creates a `pakcage.json` file that stores important information about the project itself.
- by default, we define 2 scripts in a node.js project: `start` and `test`.


> [!important] Peer Dependencies
> 
> Think of how the current module may interact with external dependents. (Other modules calling in functions, using objects, variables, et cetera.) and then see if your module depends on some other package that is required or basically is exposed in the interface the current module has with the external space. If yes, then this package is a peer dependency and should be expected by your dependent module and not your module. If not, then its a normal dependency and above dependents do not need to know about this.


## Simple Web Server
- `require("somePackage")` is almost the same thing as doing `import package` but is native to node and works much better since it is homegrown. 

>[!important] **http** module
>
>- is a built-in node module, no install needed.
>- is the low level functionality upon which Server programs like Express are built.
>- See the code below to know how it works.

To create a basic server:
```js

console.log("hello world");
const http = require('http')
const app = http.createServer((req, res) => {
	res.writeHead(200, {'content-type': 'text/plain'});
	res.write("Hello man");
	res.end("hello world");

})

  
const PORT = 3001;

app.listen(PORT);
console.log(`app is listening on port ${PORT}`);

```

## Deleting resources
```js
app.delete('/api/notes/:id', (request, response) => {
  const id = request.params.id
  notes = notes.filter(note => note.id !== id)

  response.status(204).end()
})
```

## Receiving Data
- Content-type and other headers need to be set correctly and in the right format in the body, for the server to be able to parse things correctly.
- Express's `express.json()` must be used for it to automatically `JSON.parse()` any body content that is received. 

```json
app.post('/api/notes/', (req, res)=>{

const newId = generateId();

const body = req.body;

  

if(!body.content){

console.log("entered this");

return res.status(400).json({'error': 'content not found'});

}

  

const newNote = {

content: body.content,

id: newId,

important: body.important || false

}

  

notes.push(newNote)

console.log(newNote)

res.json(newNote);

})

  

const generateId = ()=>{

  

return String(notes.length > 0? Math.max(...notes.map(note => Number(note.id))) + 1 : 0)

  

}
```
