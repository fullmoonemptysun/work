## Same origin policy & CORS

- Cross Origin Resource Sharing: Server tells browser to permit fetching resources from other origins than the server itself. 


## Application on the internet

- We will use Render as our hosting service.
For both Fly.io and Render, we need to change the definition of the port our application uses at the bottom of the _index.js_ file in the backend like so:

```js
const PORT = process.env.PORT || 3001app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`)
})
```

`process.env` is a way to access the environment variables. (key-value pairs)


`npm run build`: makes vite create the production optimized project. It creates static files that will be served as the frontend (static because they will not change or recreated like how Vite was re compiling everything every time the code changed.)

we use middleware `static` to make express serve this static content (the `index.html` and related `js` and `css` files.)

`static` inside has a convention that any `GET` requests to `/` will also look for an index.html file in the directory specified and serve that. 

Must understand that an express server is not only a server as a backend, it also serves the frontend files.

## Proxy
- There's a way in vite to set up server proxies:
```js
// https://vitejs.dev/config/
export default defineConfig({
  plugins: [react()],
  server: {    proxy: {      '/api': {        target: 'http://localhost:3001',        changeOrigin: true,      
  },    
  }  
  },
  })
```
- server: the dev server related options
- proxy: proxies for addresses
- changeOrigin: change header for the given address to the target.
