## MAC
media access control
- 48 bit unique ids assigned to NICs. 
- Local network segment (subnet or broadcast domain)
- Organizationally unique identifier.
- Ethernet frames

![[Pasted image 20260504010646.png]]

- Limited to local networks and can be spoofed.


- Randomized MAC addresses: 
	- temp. software-generated MAC
		- prevent tracking
		- improve privacy
	- How to trace?
		- DHCP logs: MAC <--> IP mappings with timestamps.
		- Signal patterns (physical layer)
		- Behavior correlation (Same DNS queries, browsing habits, device OS fingerprint)



## Layer 3 IP packets

- DHCP, Arp, Public IPs, Private IPs
- DNS, Subnetting.

- IP address $\neq$ User identity. 


## Layer 4 TCP segments

- Port number : 16 bit unsigned integer
- Ports can be used to identify services (applications in use)

## Network Packet analysis
- Header, Payload, Footer

# Network Attacks

## Denial of service
- Exhaust resource limits to block services for legit users. 
- Resource exhaustion
- Connection saturation
- Bandwidth consumption

## Ping of death 
- Send large ICMP 
- Doesn't work anymore 

## Ping flood
- overwhelm with ICMP packets.
- Doesn't work anymore

## SYN flood attack
- Flood with SYN packets without continuing the conversation and overwhelm the ports. (new port for each syn)

## Smurf 
- ICMP flood but with a IP broadcast network.

## DHCP Starvation and HTTP POST attack

- Exhaust IP address pool so nobody can connect.

- Send legit POST message ->Give content-length Long -> then send the body at a very very slow rate -> server is hung thinking it is just a slow user.


## PDoS, Login DoS, Packet Mistreating

- PDoS: Permanent Denial of service (Hardware, firmware sabotage)
- Login DoS: Spam logins with different correct usernames but wrong passwords to lock out accounts.
- Packet mistreating.
